Appendix D

The FGW Library

The following is a summary of public interface for the library provided on the CD with this book. All aspects of this library can be used without charge providing that the author's name is credited in the documentation accompanying any software that uses any part of it.

The library consists of two main sections; general purpose routines written in portable C++ which do not require any operating system support and the specific support for a mouse, keyboard and graphical window. The linker will require access to fgwlib.a if use is made of any part of the latter. All names in the fgw library are declared in namespace fgw.

General Purpose Library

The following are provided by including header file fgw_text.h into the source file. This file is self contained and does not require an implementation file nor the inclusion of a library file.

Exception Support

This header file provides two special exception types. The first one is for general library problems and the second one is specific to input problems. Readers of this book are unlikely to understand the significance of the relationship between types that is represented by derivation but that information is included so that it can be used at a later date.

problem is a general purpose simple exception class derived from std::exception. Its public interface consists of a constructor that takes a reference to a const std::string (i.e. it can take a string literal as its argument), a destructor that guarantees not to throw and a member function report() that returns a const reference to a std::string which will echo the text provided by the constructor.

A typical use would be:

throw problem("Incorrect input in fgw::xyz")

This would typically be caught and processed by

catch(problem const & p) {

     std::cout << p.report() << '\n';

}

bad_input is an exception type derived from fgw::problem specifically to report a failure in an attempt to read from an input stream. It is used by the read<T> functions to report failure. Like problem it has a single member function report() that will print out a diagnostic message.

Math Support (see chapter 5/6)

A highly accurate value for p is provided by a const double called pi. In addition two conversion functions are provided to convert degrees to radians and radians to degrees.

double const pi = 3.141592653589793;

inline double degrees(double rad){return rad*180/pi;}

inline double radians(double deg){return deg/180*pi;}

These two functions are provided to make use of the C++ trigonometric functions easier for those not familiar with the use of radian measure.

Data Input Support

There are two special functions to help with input in circumstances where some existing input may need to be skipped. There are also various other functions to provide help with input.

inline void flush_cin() (see chapter 11) first prints the warning message:

***Clearing keyboard input***

If you do not see 'DONE' press RETURN once.

because it will need help from the program user if there is no input waiting for processing. It then attempts to discard all input waiting input up to and including the next newline character. It is intended for use where the programmer knows that there may be input that must not be processed.

getdata(std::istream & in, std::string & data) (see chapter 8) is similar to std::getline() except that it skips all leading whitespace (spaces, tabs and new line characters).

Specialist input support

The following two functions handle a couple of special cases. The first uses std::cin and the second works with any std::istream object.

yn_answer() waits until the user types either 'y' or 'n' (case is disregarded). All other input is ignored. If 'y' is selected the function returns true; 'n' results in a return of false. This function requires the inclusion of fgw_text.cpp in the project.

void match(std::istream & input, char c) skips all input until the designated character is found. The next input will start immediately after the designated character.

General input support

The following set of overloaded function templates provide safe input for any type that provides an operator >> to extract data from an input stream. The first two versions provide for input from std::cin with and without a user supplied prompt. They will continue to ask for input until acceptable data (for the type being read) is received or until the maximum number of tries have been made. In the case of the latter they throw bad_input. The maximum number of tries defaults to three but can be reset by calling the function reset_max_tries() (for details see below)

From std::cin with user provided prompt:

template<typename in_type>

in_type read(std::string const & prompt)     // chapter 5/6

An example of a typical us would be:

age = read<int>("What was your age last birthday?");

From std::cin with default prompt of ': '

template<typename in_type>

in_type read()          // chapter 5/6

A typical use would be:

x = read<double>();

The third version is intended for reading from a general input stream. If the input fails it throws a bad_input exception with the message "Corrupted data stream." The assumption being that in such cases the data was not what was expected and that there is no general recovery. Of course the programmer may know better and in such circumstances can catch and handle the exception.

template<typename in_type>

in_type read(std::istream & in);             // chapter 5/6

A typical use would be:

pt1 = read<point2d>(in);

Note that if you wish to obtain input from the keyboard without a prompt you could use this last version and supply std::cin as the input stream. However this form of read<T> does not tolerate errors, it throws bad_input as soon as unsuitable data is detected.

reset_max_tries(int val) sets the value for the maximum number of tries to the value supplied and returns the old value. If the value supplied is negative it ignores it.

point2d (see chapter 5)

The definition of fgw::point2d and declarations of free functions that provide support for this type are provided by the header files point2d.h and point2dx.h. Note that these header files include fgw_text.h and so including either one will include that header automatically. Including one of these headers in your source will make the following available:

The type point2d which has the following public functionality:

·         A constructor that will create a point2d object from two doubles, one double (with the second co-ordinated defaulting to zero) and no arguments (both co-ordinates default to zero).

·         Four read access member functions x(), y(), modulus() and argument(). The first two return the Cartesian co-ordinates of the point and the second two return the polar co-ordinates.

·         Four write access member functions x(double), y(double), modulus(double) and argument(double). The first two change the corresponding Cartesian co-ordinate of the point and the second two return change the corresponding polar co-ordinate.

point2d.h also provides declarations of five functions to support input and output of point2d values to streams.

getpoint2d(std::istream & in) is a function to obtain a point2d value from an std::istream. It assumes the data will be stored as (x, y). White space is not significant but the parentheses and comma separator are.

getpoint2d() is a function that extracts data from std::cin and returns a point2d value. It assumes that the data will be provided as two numeric values without parentheses or a comma separator.

send_to(point2d pt1, std::ostream& out) streams a point2d value in Cartesian form to out. It formats the data in the form (x, y) including the parentheses and comma. It returns a reference to out.

The last two functions provide insertion (<<) and extraction (>>) operators for point2d. The extraction operator is sensitive to the stream being used so that it uses the correct form depending on whether or not std::cin is the source stream.

The following extra declarations are provided by point2dx.h. including this file automatically includes playpen.h and point2d.h. Use of the following functions will require that fgwlib.a be included in your project.

plot(playpen &, point2d, hue) is provided to allow easy plotting of point2d values in the Playpen.

distance(point2d pt1, point2d pt2) returns the distance between to points.

direction(point2d location, point2d remote) returns the direction of the remote point from the location point. Direction is measured anticlockwise from the horizontal (x-axis).

Playpen main header (see chapter 1 et al.)

The header file playpen.h provides the main declarations providing the functionality of Playpen. It starts with a number of declarations that provide various utilities and utility types to make use of Playpen easier.

Wait(unsigned ms) (see chapter 11) is a function that causes the program to wait for ms milliseconds.

hue is a type that provides fundamental facilities for managing the palette used by Playpen. Many of these facilities only make sense if you are using the default palette provided when a playpen object is created.

hue is an 8-bit type and the individual bits can be accessed by the index operator so shade[3] would give access to the third bit (remember to count from zero) of shade. E.g.

cout << shade[0] ; // output 0 or 1

shade[7] = 1;  // set the highest bit to one

shade[3] = 0;  // resets bit 3 to zero.

Attempts to access bits outside the range 0 to 7 result in returning false (0) for reading and no action for writing.

The + and  operators for hue have been overridden to provide light mixing semantics when the default palette is used. The * and / operators have been overridden so that attempts to use them will generate errors at link time. However using other operators can produce unexpected results as no comprehensive effort has been made to modify or suppress all operators.

white, black, red1, red2, red4, green1, green2, green4, blue1, blue2, blue4 and turquoise have been provided as names for hue values (chapter 1). These only make sense for the default palette. The higher the number the brighter the shade. Because only 8 bits of information are available green1, blue1 and turquoise are treated as equivalent, i.e. both green1 and blue1 are a very dim equal mixture of blue and green.

HueRGB is a supporting type to allow manipulation of the Playpen palette. Its constructor takes three char values (0 to 255) that are stored in variable r, g and b. The following three functions manage the Playpen palette (see chapter 15):

playpen& playpen::setpalettentry(hue, HueRGB const & target);

provides for changing the colour mix of a palette entry. Given that rval, gval, and bval give the new values of the red, green and blue intensities (on a scale of 0 (none) to 255 (maximum)) then:

paper.setpalettentry(32, HueRGB(rval, gval, bval));

would set entry 32 of the Playpen palette to that mixture of red, green and blue light.

HueRGB playpen::getpalettentry(hue);

provides the mechanism for recovering the current setting of a palette entry. For example:

HueRGB mix;

mix = paper.getpalettentry(19);

cout << "red intensity:   " << int(mix.r) << '\n';

cout << "green intensity: " << int(mix.g) << '\n';

cout << "blue intensity:  " << int(mix.b) << '\n';

will display the current mix of red, green and blue for palette entry 19.

playpen const & updatepalette();

activates a palette change otherwise changed palette entries will not become active until the next call of playpen::display().

colours, Xpixels and Ypixels (see chapter 8) are named integer constants that give the number of colours and the dimensions of the current implementation of Playpen. These are unlikely to change from 256, 512 and 512 in the foreseeable future.

enum plotmode {direct, additive, filter, disjoint}; // chapter 1

The nested type playpen::exception provides an exception type for playpen functions to report difficulties. This type supports a member function report() that displays an error message in the console window. There is little that you are likely to do if you get such an error other than report it to the author (and maintainer) of this book if you get it repetitively.

The nested type playen::raw_pixel_data provides for packaging the window co-ordinates of a pixel in Playpen. The member function x() and y() allow you to extract the individual x and y co-ordinate values.

The nested type playen::origin_data (chapter 5/6) provides for packaging the window co-ordinates of a playpen origin. The member function x() and y() allow you to extract the individual x and y co-ordinate values.

The origin used by a playpen object is managed by:

playpen& playpen::origin(int x, int y);  // chapter 5/6

changes the origin to the raw window co-ordinates specified so that (x, y) in window co-ordinates becomes (0, 0) in Playpen ones.

origin_data playpen::origin();

returns the present playpen's origin in raw windows co-ordinates. For example:

playen::origin_data orig(paper.origin());

cout << "The origin of paper is ("

      << orig.x() << ", " << orig.y() << ")\n";

will display the current position of paper's origin in raw windows co-ordinates. The default value for this is (256, 256).

The scale of a playpen which determines the size of a logical pixel is managed by:

bool playpen::scale(int i);    // chapter 1

which attempts to change the scale to i (i.e. make a logical pixel i by i physical pixels). If the attempt is unsuccessful because i is greater than 64 or less than 1 the function returns false otherwise it returns true.

int playpen::scale();

returns the current scale of the playpen object.

The following function provides management for the plotting mode property of a playpen.

plotmode playpen::setplotmode(plotmode pm); // chapter 1

both sets a new mode and returns the old one. This function combines these two tasks because you will want keep the current plotting mode when changing to an alternative one so that you can restore it when you have finished the current task. A typical use is:

plotmode const oldmode(paper.setplotmode(disjoint));

// do something

paper.setplotmode(oldmode); // restore old state

The following pair of functions provide the basic plotting functionality:

playpen& playpen::plot(int x, int y, hue h);  // chapter 1

playpen& playpen::plot(double x, double y, hue h);  // chapter 5/6

The second version is to suppress warning messages when floating-point co-ordinates are used. Those two functions take account of the current origin and scale. The following function provides a plotting facility at raw windows co-ordinate level:

void playpen::setrawpixel(int x, int y, hue h);

where x and y are in raw windows co-ordinates (from (0, 0) top left to (512, 512) bottom right).

playpen::display();  // chapter 1

forces the Playpen to display all changes that have been made since the last time it was updated.

The following two functions allow recovery of the current hue of a pixel:

hue playpen::get_hue(int x, int y);     // chapter 10

hue playpen::getrawpixel(int x, int y);  // chapter 9

The first uses Playpen co-ordinates and returns the value of the bottom left physical pixel of a playpen logical pixel (i.e. allowing for scale and origin). The second function deals with raw window pixels and (x, y) are measured in windows terms (x measure to the right of the left edge and y measure down from the top edge).

playpen& playpen::clear(hue shade = white);  // chapter 1

sets all the pixels of Playpen to shade (default 255, which may not be white if the current palette is not the default one).

playpen& playpen::rgbpalette();              // chapter 15

restores the default palette.

raw_pixel_data playpen::get_raw_xy(int i, int j);

returns the raw co-ordinates of the physical pixel that is the bottom left of the logical allowing for scale and origin) identified by (i, j).

playpen::playpen(hue background = white);  // chapter 1

constructs a playpen object and clears the Playpen to background if it also creates the Playpen. However if the Playpen already exists the background hue is ignored.

Mouse support (chapter 11)

Including mouse.h provides minimal support for using a mouse in Playpen. All mouse objects share the same data object, which is the underlying Playpen object. Creating a mouse object creates the Playpen if it does not yet exist.

The mouse type has two member functions (cursor_at() and button_pressed()) and a nested type (mouse::location) used by cursor_at().

mouse::location has x() and y() member functions that return the x and y values currently stored in a mouse::location object.

cursor_at() returns a mouse::location value storing the raw window co-ordinates of the mouse cursor if Playpen has the focus and the mouse cursor is in the Playpen. Otherwise it returns a mouse::location value of (-1, -1).

button_pressed() returns true if a mouse button is pressed and Playpen has the focus, otherwise it returns false.

See chapter 11 for functions playpen_x() and playpen_y() that convert raw window values to Playpen co-ordinates that take account of the origin and scale of a playpen object. These functions are not part of the fgw library.

Portable Network Graphics support (see chapter 9)

My library includes rather more support for PNG than has been detailed in this book. However the extra functionality (declared in minipng.h) requires considerable expertise and so will not be detailed here though readers can study this header file if they wish.

The two functions that have been used in this book are declared in playpen.h. They are: LoadPlaypen() and SavePlaypen().

LoadPlaypen(playpen & paper, std::string filename) loads paper from the file designated by filename. If the file fails to open a minipng::error is thrown. If the file is unsuitable for use a playpen::exception is thrown. The loaded image requires a call to playpen::display() to make it visible.

SavePlaypen(playpen const & paper, std::string filename) saves paper to the file designated by filename. If the file fails to open a minipng::error is thrown.

Drawing support

Two header files provide support for line drawing; line_drawing.h provides the declarations for drawing various straight lines, and via plot_policy parameters other uses of moving from a start point to an end point. The header file shape.h provides support for drawing and manipulating shapes composed of straight-line segments.

Line drawing (see chapters 3, 5/6 and 15)

The functions declared in line_drawing.h break into two main groups, those designed for drawing vertical and horizontal lines and those designed for drawing general lines. Each set consists of a number of overloads to deal with different data.

The following two entities are also declared in this header file:

typedef void (plot_policy)(playpen &, int x, int y, hue) which declares plot_policy as a function type suitable for use with functions requiring some form of plotting behaviour.

plot(playpen & p, int x, int y, hue shade = black) which declares plot_policy type function that delegates its action to p.plot(x, y, shade). The default parameter allows it to be used without specifying a hue.

The vertical and horizontal line drawing functions are:

vertical_line(playpen &,

     int hpos, int y1, int y2, hue, plot_policy plotter = plot);

horizontal_line(playpen &,

     int vpos, int x1, int x2, hue, plot_policy plotter = plot);

These two functions are the basic functions for drawing vertical and horizontal lines. The default plot_policy argument means that when they are called without a user provided plot_policy function they will use the playpen::plot() function delegated by a free function plot() given above.

vertical_line(playpen &,

        int hpos, int y1, int y2, plot_policy plotter = plot);

horizontal_line(playpen &,

        int vpos, int x1, int x2, plot_policy plotter = plot);

The above versions allow drawing of vertical lines and horizontal lines with a default colour (of black) even when the programmer is providing a plot_policy function.

vertical_line(playpen &,

     point2d, int length, hue shade, plot_policy plotter=plot);

horizontal_line(playpen &,

     point2d, int length, hue shade, plot_policy plotter=plot);

The above provide vertical and horizontal line drawing using a point2d starting point and a length with a specified colour.

vertical_line(playpen &,

        point2d, int length, plot_policy plotter=plot);

horizontal_line(playpen &,

Support for drawing shapes (see chapters 5/6 and 8)

The declarations of fourteen functions and a typedef that provide support for some simple shape creation, manipulation and drawing are provided by shape.h.

typedef std::vector<point2d> shape;

provides a convenient name for the container of vertices that define a shape.

void filled_polygon(playpen &,

             shape const &, point2d centre, hue shade);

void filled_polygon(playpen & pp, shape const & s, hue shade);

provide for drawing a filled polygon either by specifying an internal point or by computing an internal point from the vertices. Both functions assume that the shape is a simple closed shape and that the given or computed internal point is indeed internal to the shape. If the point is external the exterior will be filled. If the point is on the boundary the result is unspecified.

void drawshape(playpen & pp, shape const & s, hue shade);

draws a shape by drawing straight lines between the vertices provided.

void moveshape(shape & s, point2d offset);

adjusts the vertices stored in s by the amount specified by the offset.

void growshape(shape & s, double xfactor, double yfactor);

multiplies the x and y co-ordinates of the vertices stored in s by the xfactor and yfactor.

void scaleshape(shape & s, double scalefactor);

similar to growshape() but uses the same factor for both co-ordinates.

void rotateshape(shape & s, double rotation, point2d centre);

void rotateshape(shape & s, double rotation);

adjusts the vertices stored in s so as to rotate a shape about a centre or the origin by the amount given by rotation.

void sheershape(shape & s, double sheer);

adjust a shape by sheering it parallel to the x-axis.

double area_of_triangle(shape s);

computes the area defined by the first three vertices stored in s. It returns zero if there are fewer than three vertices in s.

shape make_regular_polygon(double radius, int n);

returns a shape that contains the vertices of a regular n-sided polygon centred at the origin with the first vertex on the x-axis.

shape makecircle(double radius, point2d centre);

returns a shape object with a sufficiently many sided polygon to approximate to a circle at screen resolution.

shape read_shape(std::istream &);

void write_shape(shape const & s, std::ostream &);

these last two functions provide for saving and recovering shape data from a stream.

Filling areas (see chapter 8)

flood_fill.h declares two functions that are designed to fill areas with a single colour. Including this header automatically also includes playpen.h. Your project will need to include fgwlib.a if you use either of the fill routines. Note that both these routines take Playpen co-ordinates but actually fill at raw pixel level so for the purposes of selecting pixels the scale of the playpen object is ignored.

seed_fill(playpen & paper, int i, int j, hue new_shade, hue boundary) changes all pixels within a boundary provided by the boundary hue with new_shade. The edges of the Playpen are automatically part of the boundary.

replace_colour(playpen & canvas, int i, int j, hue new_shade) changes a contiguous area (orthogonally connected) whose colour is that of the seed pixel to the new_shade.

Named colour support

The colournames.h header file provides an extension of the colour names provided by playpen.h to include a range of mixtures and intensities. These names are only appropriate if the default palette is being used and in many cases only approximate to the name given. The higher the number in a name the brighter (more intense) the shade will be.

In general x3 is equivalent to x1+x2, x5 is equivalent to x1+x4, x6 is equivalent to x2+x4 and x7 is equivalent to x1+x2+x3 where x is replaced by a colour from red, green, blue, magenta, yellow and cyan.

The added colour names are:

red3, red5, red6, red7

green3, green5, green6, green7

blue3, blue5, blue6, blue7

yellow1, yellow2, yellow3, yellow4, yellow5, yellow6, yellow7

magenta1, magenta2, magenta3, magenta4, magenta5, magenta6, magenta7

cyan1, cyan2, cyan3, cyan4, cyan5, cyan6, cyan7

grey1, grey2, grey3, grey4, grey5, grey6

Looking at this header file will show you how to add other colour names for yourself.
